What I changed
I’ve updated the template so that markers stop clustering around New York by default and instead follow your chosen destination (like Paris) more closely.

1. Service markers now bias around the destination, not NYC
In client/src/pages/experience-template.tsx, the mapProviders calculation changed from always centering service markers near NYC to:

ts
const mapProviders = useMemo(() => {
  // Only include provider services on the map after the user submits details,
  // to avoid confusing default clusters.
  const serviceMarkers = detailsSubmitted
    ? filteredServices.map((<span class="hljs-params">s, index) =></span> {
        const numericId = typeof s.id === "number" ? s.id : parseInt(String(s.id), 10) || index;
        const baseHash = numericId * 1000;
        const latOffset = ((baseHash % 100) - 50) / 1000;
        const lngOffset = (((baseHash + 37) % 100) - 50) / 1000;

        return {
          id: s.id.toString(),
          name: s.serviceName,
          category: s.serviceType || currentTabCategory || "venue",
          price: Number(s.price) || 0,
          rating: Number(s.averageRating) || 4.5,
          // Bias around the geocoded destination center when known;
          // otherwise fall back to the previous NYC-ish placeholder.
          lat: destinationCenter?.lat ?? 40.7128 + latOffset,
          lng: destinationCenter?.lng ?? -74.006 + lngOffset,
          description: s.shortDescription || s.description || undefined,
        };
      })
    : [];
  // ...customVenueMarkers, hotelSearchMarkers, activitySearchMarkers...
  return [...customVenueMarkers, ...serviceMarkers, ...hotelSearchMarkers, ...activitySearchMarkers];
}, [
  filteredServices,
  currentTabCategory,
  customVenues,
  activeTab,
  hotelSearchMarkers,
  activitySearchMarkers,
  detailsSubmitted,
  destinationCenter?.lat,
  destinationCenter?.lng,
]);

Key effects:

Before submitting details: no generic provider markers are shown (so you don’t see a random NYC cloud when you first open the template).
After you submit a destination (e.g., “Paris”):
destinationCenter is computed from /api/geocode.
Service markers are distributed with small offsets around destinationCenter, so they appear near Paris instead of New York.
Custom venues, hotel search results, and activity search results still use their real coordinates.
2. What you should see now
Open the Travel template with a fresh page load:
The map should no longer show a dense NYC pin cluster by default.
Until you submit a destination, it will primarily show things that have actual coordinates (e.g., custom venues / search markers) or just the default map state.
Enter “Paris” as the destination and click “Submit Travel Details”:
The map centers to Paris (via existing destinationCenter logic).
Any auto-generated service markers will appear near that Paris center (instead of NYC).
Hotel/activity/custom-venue markers for Paris will appear exactly where their coordinates say they are.
