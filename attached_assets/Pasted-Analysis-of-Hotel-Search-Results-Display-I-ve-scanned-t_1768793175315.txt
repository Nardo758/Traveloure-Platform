Analysis of Hotel Search Results Display
I’ve scanned the HotelSearch component, where the hotel search results are fetched, filtered, and rendered.

Likely Root Causes
Results silently filtered out (common “not displayed properly” cause)
The component never renders anything unless filteredAndSortedHotels.length > 0. However, filteredAndSortedHotels is derived with hard filters:

ts
let result = hotels.filter((<span class="hljs-params">hotel) =></span> {
  const offer = hotel.offers?.[0];
  if (offer) {
    const price = parseFloat(offer.price.total);
    if (price > maxPrice) return false;  // drops hotels above maxPrice
  }
  
  if (starRating > 0 && hotel.hotel.rating) {
    const hotelStars = parseInt(hotel.hotel.rating);
    if (hotelStars < starRating) return false; // drops hotels below starRating
  }
  
  return true;
});

If maxPrice or starRating props are set too aggressively by the parent, all hotels may be filtered out even though the backend returned data, leading to no cards rendering.
There is no visible UI for maxPrice / starRating in this component, so the user can’t see or adjust these constraints from the hotel results screen.
No explicit “no filtered results” state

If the API returns hotels but every one is filtered out, the user sees nothing under “loading” and “error” blocks; the hotels && hotels.length === 0 branch only covers the case where the API returned an empty array, not the case where hotels.length > 0 but filteredAndSortedHotels.length === 0.
This strongly matches a user perception of “results aren’t being displayed properly.”
Potential mis‑wiring of props from parent

HotelSearch expects destination, checkIn, checkOut, etc. from a parent page, which you didn’t include here. If the parent:
passes maxPrice={0} or some unintended default, or
passes starRating={5} always, then you will almost always filter everything out.
Also, canAutoSearch requires all of cityCode, checkInDate, checkOutDate, and adults to be truthy. If any is missing, the query is disabled and no results are fetched.
Minor bug in the card rendering for proximity
In the render loop you use hotel.latitude and hotel.longitude:

ts
const proximityScore = hotel.latitude && hotel.longitude && activityLocations.length > 0
  ? getProximityScore(hotel.latitude, hotel.longitude, activityLocations)
  : null;
but the HotelOffer type defines latitude/longitude on hotel.hotel, not on hotel itself. This will always evaluate falsey and skip proximity info, though it doesn’t stop cards from rendering.

What I Recommend Fixing
To fix the “not displayed properly” behavior:

Add a clear empty‑filtered state so users see a message when filters hide all hotels.
Loosen or expose maxPrice and starRating (or ensure parent passes realistic defaults).
Fix the proximity score bug (hotel.hotel.latitude / hotel.hotel.longitude).
